# v1.1.4 - Install Flow Hardening And Codex Plugin Surface

**Status:** patch release. The core workflow loop and provider support posture are unchanged.

## Highlights

- **Install-first CLI** - the redundant `init` command is removed; `install` is the single supported project bootstrap path.
- **Non-git and provider UX** - project installs outside git repos get a clearer fallback path, and provider prompts respect binary availability again.
- **Codex plugin surface** - native Codex install now ships agents, hooks, domain generation, and a build workflow for the plugin package.
- **CI reliability** - lint, Node 18 wizard, and Windows smoke-install regressions are fixed across the main CI matrix.

## What changed

### Added

- Codex plugin agents (`explorer`, `fixer`, `gatekeeper`, `reviewer`, `verifier`) and `hooks.json` with a Codex hook router.
- `scripts/build-codex-plugin.js` and a Codex plugin build workflow.
- Expanded Codex-native install surface: domain skill generation, rule schema generation, and plugin packaging helpers.
- ESLint flat config and dev-dependency updates for supported CI legs.

### Changed

- `install` is now the only bootstrap command; use `npx ai-engineering-harness install` instead of `init`.
- Non-git project installs now warn and continue with a private-surface fallback instead of hard-stopping.
- Provider selection prompts gate on detected provider binaries again after the install flow refactor.
- `flow` command routing and install command allow-list behavior are aligned with the current CLI surface.

### Removed

- `init` command and its dedicated test surface. It was a non-interactive wrapper around `install` that bypassed user prompts.

### Fixed

- Install wizard regressions on Node 18 and Windows smoke-install paths.
- CI lint leg scope, `@clack/prompts` and `@types/node` dependency drift, and test references to the removed `init` command.
- Codex rule schema generation and domain generation coverage gaps.

## The loop is unchanged

```text
Session Start → Discuss → Plan → Run → Verify → Ship → Remember
```

## Verification

- `npm run build`
- `npm test`
- `node bin/validate.js`

## Upgrade notes

- Replace `npx ai-engineering-harness init` with `npx ai-engineering-harness install` in scripts and docs.
- No migration is required for existing `.harness/` or `.ai-harness/` install surfaces.
- Codex users can continue npm project install; the expanded plugin surface is additive until marketplace publish is complete.
