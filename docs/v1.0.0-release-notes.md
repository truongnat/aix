# v1.0.0 — Workflow Guardrails Foundation

**Status:** stable contract for dogfooding and public adoption. Not a claim of full autonomous runtime enforcement on every provider.

## Product identity

`ai-engineering-harness` is a markdown-first engineering workflow guardrail kit for AI coding agents.

It is **not** an agent framework, heavy runtime platform, universal slash-command system, or replacement for human review.

## Highlights

- **Agent System Prompt** — provider-neutral senior engineering behavior, MUST/MUST NOT rules, blocked output, evidence standards, and response formatting ([agent-system/SYSTEM_PROMPT.md](../agent-system/SYSTEM_PROMPT.md))
- **Session Start protocol** — `harness-start` as mandatory boot sequence ([docs/session-start.md](session-start.md))
- **Canonical command loop** — hyphen-form IDs across map → remember
- **Provider adapters** — honest Claude native path; Cursor/Codex/Gemini fallbacks
- **Session-based memory** — root `.harness/` router; `sessions/` working artifacts
- **Blocking protocol** — structured `### Blocked` when preconditions fail
- **Dispatch prompt templates** — plan, run, verify, ship with guard branches
- **Tool discovery and routing** — `scripts/discover-tools.js` + capability docs
- **Hooks** — phase guard, evidence recording, memory compaction (portable core scripts)
- **Dynamic skills** — create, compose, archive/dispose lifecycle
- **Delegated workers** — Claude `.claude/agents/harness-*` adapter path
- **Daily dev report** — `REPORT.md`, `PR_MESSAGE.md`, `CHANGE_SUMMARY.md` on ship
- **Validation** — layered `lib/validate/` contracts (421+ checks)
- **GitHub Pages** — 1.0.0 landing page at [truongnat.github.io/ai-engineering-harness](https://truongnat.github.io/ai-engineering-harness/)

## Honest limitations

- Claude Code has the strongest native path (project commands, agents, hooks examples).
- Cursor, Codex, and Gemini use fallback/adapted rule surfaces.
- Hooks are provider-specific; not identical across all agents.
- Optional local tools are best-effort.
- The harness provides guardrails and artifacts — not guaranteed correctness.

## Upgrade from 0.11.x

- Treat **Session Start** as the entry ritual before plan/run/verify/ship.
- Use session paths under `.harness/sessions/<id>/` for working artifacts.
- Expect `harness-ship` to produce report artifacts when verification allows.
- Continue using hyphen-form command IDs only.
