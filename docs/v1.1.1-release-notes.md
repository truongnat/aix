# v1.1.1 - Headless Domain Bootstrap And Release-Facing Cleanup

**Status:** patch release. No breaking changes to command IDs, provider support, or the core workflow loop.

## Highlights

- **Headless domain bootstrap** - `aih domains` now accepts `--analysis-file` or stdin, letting the agent generate domain skills non-interactively.
- **Non-interactive init/install** - `init` and `install` no longer force domain prompts; fresh init defaults to `domains: []` unless the caller passes explicit domain input.
- **Session-start guidance** - session bootstrap docs now require an explicit visible "analyzing the codebase" message before automatic domain analysis.
- **Release-facing version sync** - the release surfaces now track the current package version and the new patch notes file.

## What changed

### Added

- `lib/cli-commands/domains.ts` - headless domain-skill generation command for file or stdin input.
- CLI dispatch and help updates for `domains`.
- Session-start and prompt-template guidance for visible domain analysis bootstrap.

### Changed

- `init` now keeps domain selection deterministic and non-interactive unless explicit domain input is provided.
- `install` no longer forces the interactive domain picker.
- `README.md` and `docs/README.md` now point at the `v1.1.1` release surfaces.

## The loop is unchanged

```text
Session Start → Discuss → Plan → Run → Verify → Ship → Remember
```
