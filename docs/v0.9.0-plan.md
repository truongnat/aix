# v0.9.0 Plan

## Purpose

Plan `v0.9.0` as a Stable Contract Freeze release that locks the core contracts required for `v1.0.0` without adding new product features or distribution automation.

## Target Summary

`v0.9.0` should make the harness capability pack contract explicit, indexed, and stable enough that `v1.0.0` can ship as the first stable release with predictable breaking-change rules.

## Scope

- define stable contract index
- freeze `PACK.md` heading contract
- freeze installed surface contract
- freeze target profile artifact contract
- freeze goal artifact contract
- freeze `validate.js` structural behavior
- freeze runtime consumption boundaries
- freeze manual packaging flow
- document breaking-change policy
- classify minimal install tier as post-v1 optional (documented decision)

## Non-Goals

- no new automation
- no runtime adapters
- no package or marketplace publishing
- no semantic validation
- no archive generation automation
- no minimal install tier implementation
- no Scenario C dogfood requirement
- no contract changes driven only by source-repo polish without v1 alignment

## Release Risks

- freezing docs without aligning `validate.js` behavior creates false stability
- introducing install tiers during v0.9 would expand contract surface before v1
- breaking-change policy too vague to guide v1.0.0 contributors

## Safety Boundaries

- markdown remains the source of truth
- structural validation only
- no secrets in contract artifacts
- contract freeze is documentation and behavior alignment, not new runtime systems
