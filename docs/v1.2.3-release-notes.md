# v1.2.3 Release Notes

## Stack Scanner — framework detection and domain inference

A new `stack-scanner` module detects frameworks, languages, and infrastructure from project files and infers which harness domains apply.

The scanner reads package manifests, config files, and directory structure to produce a `StackScanResult` with detected frameworks grouped by domain (frontend, backend, devops, etc.).

### New CLI command: `harness scan`

```bash
npx ai-engineering-harness scan
```

Runs the stack scanner against the current project and prints a summary of detected frameworks and inferred domains. Results are written to `.harness/config.json` under the `stackMeta` key.

### `domains` command auto-scans

The `harness domains` command now automatically runs a stack scan before listing domains. Stack metadata is merged into `.harness/config.json` so domain skills have accurate context about the project's technology stack.

## Bug fix: non-shell tools in Codex hook router

The Codex hook router previously crashed when processing tool events for non-shell tools that have no `command` field (e.g., Read, Write, Edit). The router now guards against missing `command` fields and passes these events through without error.

## Files changed

- `lib/stack-scanner.ts` — new module: framework detection via file evidence accumulation
- `lib/stack-detect.ts` — `StackScanResult` type and `mergeStackSignals()` utility
- `bin/aih.js` — `scan` command registered in CLI
- `lib/commands/scan.ts` — `scan` command implementation
- `lib/commands/domains.ts` — auto-scans before listing domains, writes `stackMeta` to config
- `hooks/core/codex-hook-router.js` — guard against tools with no `command` field
