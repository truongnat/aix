# v0.6.0 Readiness

## Purpose

Audit whether `v0.6.0` is ready to release as a docs-only Runtime Consumption Guides release.

## Target Summary

`v0.6.0` explains how common AI coding runtimes should consume the plugin-like markdown capability pack inside a target repository.

The release reframes runtime docs around capability-pack consumption, shared prompt patterns, and structural validation flow without adding adapters, integrations, or packaging automation.

## Completed Capabilities

- runtime consumption model in `docs/runtime-consumption-model.md`
- runtime docs index update in `docs/runtimes/README.md`
- Claude Code capability pack consumption guide
- Cursor capability pack consumption guide
- Codex capability pack consumption guide
- Gemini CLI capability pack consumption guide
- OpenCode capability pack consumption guide
- runtime comparison guide in `docs/runtimes/comparison.md`
- shared prompt and validation patterns documented in the comparison guide and per-runtime guides

## Remaining Gaps

- no runtime adapters
- no runtime integrations
- no marketplace package
- no package publishing automation
- no semantic validation
- runtime docs may need future updates if tools change

## Release Blockers

No release blockers identified.

The intended `v0.6.0` scope is docs-only runtime consumption guidance, and that surface is now present and internally consistent with `docs/v0.6.0-plan.md`.

## Non-Blockers

These are intentional non-blockers for `v0.6.0`:

- no runtime adapters
- no runtime integrations
- no marketplace package
- no package publishing automation
- no semantic validation
- runtime docs may need future updates if tools change

These are outside the release target because `v0.6.0` documents usage guidance rather than implementing runtime-specific code or distribution automation.

## Validation Status

Current validation status:

- `node bin/validate.js` passes
- target profile fixture validation passes
- target goal fixture validation passes

Validation remains structural-only. Runtime guides do not introduce new runtime behavior.

## Test Status

Current test status:

- `npm test` passes
- install surface tests remain in place
- target profile and target goal validation coverage remains in place

## Safety Review

- markdown remains the source of truth
- no dependencies were added
- no `src/` directory was added
- no runtime adapters or integrations were added
- no marketplace or package-publishing automation was added
- no server, database, or background systems were added
- no semantic validation or deep scanning was added
- runtime guidance stays docs-only and preserves source pack vs target repo boundaries

## v0.6.0 Release Decision

`v0.6.0` is ready to release as a docs-only Runtime Consumption Guides release.

It makes the shared consumption pattern explicit across Claude Code, Cursor, Codex, Gemini CLI, and OpenCode without expanding into adapters, integrations, or packaging automation.
