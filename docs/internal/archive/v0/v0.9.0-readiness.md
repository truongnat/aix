# v0.9.0 Readiness

## Purpose

Audit whether `v0.9.0` is ready to release as the Stable Contract Freeze release on the path to `v1.0.0`.

## Target Summary

`v0.9.0` freezes the core capability pack contracts adopters and maintainers can rely on before the first stable `v1.0.0` release. It does not add distribution automation, runtime adapters, or new product features.

## Remaining Gaps

Intentional gaps after contract freeze (not missing work for `v0.9.0` scope):

- no archive generation automation
- no package or marketplace publishing automation
- no runtime adapters or plugins
- no semantic validation or secret scanning
- no minimal install tier before v1.0.0
- no Scenario C dogfood requirement
- `PACK.md` body and version values not validated in code
- example files may remain richer than minimum heading contracts (layout guide covers adoption)

## Release Blockers

**No release blockers identified.**

All eight core contracts are documented and indexed. Source pack validation and target fixture validation pass. Dogfood evidence from `v0.8.0` remains valid for v1 path.

## Non-Blockers

These are intentional non-blockers for `v0.9.0`:

- automation gaps listed above (v0.9.0 freezes contracts, not tooling)
- Scenario C not executed
- minimal install tier not implemented
- `PACK.md` still at `v0.8.0` version string until release pass updates it
- no `v0.9.0-release-notes.md` until release pass (readiness is pre-release)

`v0.9.0` requires contract documentation and audit clarity, not new dogfood or adapters.

## Safety Review

- markdown remains the source of truth

- no runtime adapters, server code, or distribution automation added
- frozen contracts preserve source-pack vs target-repo boundary
- breaking-change policy documents pre-v1 and post-v1 expectations

## v0.9.0 Release Decision

**`v0.9.0` is ready to release** as Stable Contract Freeze after the release pass (`PACK.md` version, release notes, dated CHANGELOG, README badge).

Next milestone: **`v1.0.0` Stable Capability Pack Release** — finalize stable release using frozen contracts without expanding contract surface unless a blocker appears.
