# v0.9.x Readiness

## Purpose

Decide whether the current `main` branch (runtime-native installer + dogfood evidence) is ready for an **experimental v0.9.x** release—not a stable `v1.0.0` runtime guarantee.

Complements [v0.9.0-readiness.md](v0.9.0-readiness.md) (contract freeze) and [runtime-dogfood-summary.md](runtime-dogfood-summary.md) (dogfood evidence).

## Target Summary

`v0.9.x` delivers a **runtime- and scope-aware** installer that writes bootstrap files under runtime-correct paths, initializes project-local `.harness/`, and validates targets with optional `--runtime`. Adoption UX pivots away from copying the full pack into product repo roots.

## Runtime Dogfood Evidence

Full matrix: [runtime-dogfood-summary.md](runtime-dogfood-summary.md).

All six primary runtimes dogfooded for **project** file/install (plus gemini global write, claude global dry-run). **No** runtime has stable manual session proof.

## Installer UX State

- **Recommended:** `sh install.sh --runtime <name> --scope project|global [--init-harness] [--yes]`
- **Validate:** `node bin/validate.js --target <repo> --runtime <name> --profile-only` (from source pack)
- **Fallback:** `--runtime manual` or default non-interactive `curl | sh` → root copy + warning
- **Not recommended:** `--runtime all` without per-runtime verification

## Validation State

- Pack self-validation: passing (required docs include dogfood reports).
- Legacy target profile: `AGENTS.md` + `.harness/`
- Runtime-aware target profile: bootstrap paths per runtime + `.harness/`
- No semantic validation; no plugin-installed detection

## Remaining Gaps

- Manual IDE/CLI sessions for most runtimes
- `--runtime all` not dogfooded
- Claude marketplace plugin install manual step
- Gemini project-local load uncertain
- README/install next-steps polish (absolute paths, legacy validate hints)
- Optional: dedicated `valid-target-profile-opencode` fixture (tests use temp dirs today)

## Release Blockers

**None identified for an experimental v0.9.x release** if documentation clearly states:

- runtime-native modes are **experimental**
- stable per-runtime support is **No**
- manual fallback remains available

**Blockers for v1.0.0 stable runtime claims:**

- Manual runtime verification per runtime (or reduced v1 scope)
- Policy for global config UPDATE on existing machines
- Decision on `--runtime all`

## Non-Blockers

- Codex manual check blocked by API quota (does not invalidate file/install evidence)
- Legacy `--profile-only` fail without `AGENTS.md` on non-AGENTS runtimes (documented; use `--runtime`)
- Gemini `extensions list` empty output
- Claude global write skipped on dogfood host (safe practice)
- No npm publish / marketplace automation (explicitly out of scope)

## Safety Review

- No root pollution in runtime-native dogfood
- Global writes documented with dry-run-first guidance
- No secrets in dogfood reports
- `--init-harness` rejected for global scope (no shared `.harness`)

## Release Decision

**Recommendation: v0.9.1 may ship as experimental runtime-native installer** (tag `v0.9.1`).

**v0.9.2 should address real-install UX and Git hygiene before wider promotion** — private mode via `.git/info/exclude` (not default `.gitignore`), install wizard, uninstall/update, Antigravity research. See [installer-ux-v0.9.2-plan.md](installer-ux-v0.9.2-plan.md).

Conditions for **continued** experimental promotion:

1. [v0.9.x-release-scope.md](v0.9.x-release-scope.md) and [runtime-dogfood-summary.md](runtime-dogfood-summary.md) remain accurate.
2. [README.md](../README.md) and [plugin-install-ux.md](plugin-install-ux.md) do not claim stable runtime support.
3. After `v0.9.2`: private/shared visibility documented; `.git/info/exclude` for private; `.gitignore` only explicit.

**v1.0.0** should **not** claim stable runtime-native support until selected runtimes pass manual session criteria documented in the audit.

Manual fallback (Scenario C) remains supported as **fallback only**, not the long-term default UX.
