# v0.4.0 Release Notes

## Summary

`v0.4.0` is an ergonomics-polish release for first-time harness adoption.

It improves install-to-profile clarity, tiny-repo onboarding, and structural validation guidance without adding runtime complexity.

## What Is Included

- post-install next-step guidance
- install output examples
- fixed installed docs surface for post-install guidance
- tiny repo adoption example
- small repo memory conventions
- compact small memory template
- install-to-profile walkthrough
- validation troubleshooting guide
- `v0.4.0` readiness and release scope docs

## What Changed From v0.3.1

`v0.3.1` documented dogfood findings and pointed toward ergonomics follow-up work.

`v0.4.0` turns that direction into adoption-facing improvements:

- clearer post-install guidance after `install.js`
- better examples for expected install output
- installed docs that match the post-install guidance path
- a compact tiny-repo example that shows profile and goal artifact shape
- lighter memory guidance for very small repositories
- a short walkthrough from install to validated profile and goal artifacts
- a troubleshooting guide for common structural validation failures

## What Is Intentionally Not Included

- runtime adapters
- automatic harness or profile generation
- interactive setup wizard
- semantic validation
- secret scanning
- deep repository scanning
- server, database, or background systems
- package publishing automation

## Who Should Use This Release

Use `v0.4.0` if you want:

- a clearer first-time adoption path
- better guidance after running `install.js`
- a tiny reference repo shape for profile and goal artifacts
- lighter memory guidance for small repositories
- structural validation help without broadening validator scope

## How To Use The Improved Install-To-Profile Flow

Use the current flow in this order:

```bash
node bin/aih.js install --target ../my-project --dry-run
node bin/aih.js install --target ../my-project
node bin/validate.js --target ../my-project --profile-only
node bin/validate.js --target ../my-project --goal health-check
```

Then read:

- [docs/install-output-example.md](install-output-example.md)
- [docs/install-to-profile-walkthrough.md](install-to-profile-walkthrough.md)
- [docs/validation-troubleshooting.md](validation-troubleshooting.md)
- [docs/small-repo-memory.md](small-repo-memory.md)

## Known Limitations

- validation remains structural-only
- installer output can still feel dense on large copy surfaces
- no interactive setup wizard exists
- no automatic profile generation exists
- passing validation does not prove application correctness

## Next Likely Milestone

The next likely milestone is a `v0.4.x` or later follow-up that reviews whether more ergonomics polish is needed before any optional automation ideas are considered.
