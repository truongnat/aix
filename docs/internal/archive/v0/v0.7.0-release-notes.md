# v0.7.0 Release Notes

## Summary

`v0.7.0` is a Capability Pack Packaging release.

It defines a lightweight `PACK.md` manifest, manual packaging contract, pack verification checklist, and structural `PACK.md` heading validation so the harness can be identified and distributed as a versioned capability pack without archive or publishing automation.

## What Is Included

- `PACK.md` manifest
- pack manifest spec
- packaging model
- pack verification checklist
- lightweight `PACK.md` heading validation
- manual packaging guide
- release archive model and checklist integration
- installed surface alignment docs
- `v0.7.0` readiness and release scope docs

## What Changed From v0.6.0

`v0.6.0` documented how common runtimes consume the capability pack in a target repository.

`v0.7.0` adds packaging contracts on top of that model:

- a markdown-first `PACK.md` manifest for pack identity, surface, and validation commands
- a packaging model and manifest spec
- a maintainer pack verification checklist
- `node bin/validate.js` checks for required `PACK.md` section headings
- a manual packaging guide for release and optional archive assembly
- clearer rules for when `PACK.md` belongs in source pack and release archives versus target repos

## What Is Intentionally Not Included

- archive generation automation
- npm or package publishing
- marketplace automation
- runtime adapters
- semantic manifest body validation
- installed surface checksums
- server, database, or background systems
- deep repository scanning

## Who Should Use This Release

Use `v0.7.0` if you:

- maintain or distribute the harness as a versioned capability pack
- need a manifest and checklist before manual release or archive distribution
- want structural `PACK.md` validation without adding package managers or build automation
- already use `v0.5.0` distribution and `v0.6.0` runtime consumption docs

## How To Verify The Pack

From the source pack repository root:

```bash
node bin/validate.js
npm test
node bin/validate.js --target test/fixtures/valid-target-profile --profile-only
node bin/validate.js --target test/fixtures/valid-target-goal --goal google-login
```

`node bin/validate.js` checks required `PACK.md` headings. It does not validate manifest body content, version semantics, or archive file lists.

Complete [pack-verification-checklist.md](pack-verification-checklist.md) before release for manual consistency checks.

## How To Package Manually

Follow [manual-packaging-guide.md](manual-packaging-guide.md):

1. confirm release scope
2. set **Pack Version** in `PACK.md` to the releasing version
3. run the pack verification checklist
4. run validation commands
5. prepare release notes and tag the source repository
6. optionally assemble a release archive using [release-archive-checklist.md](release-archive-checklist.md)

Do not expect an archive generation script in this release.

## Known Limitations

- no archive generation automation exists
- no npm, marketplace, or package publishing automation exists
- manifest validation is heading-only
- `PACK.md` is not required in every target repository by default
- manual archive assembly remains maintainer-driven

## Next Likely Milestone

The next likely milestone is optional follow-up work: archive generation scripts, manifest body or version checks, installed surface checksums, or marketplace packaging—only if the project explicitly expands beyond manual packaging contracts.
