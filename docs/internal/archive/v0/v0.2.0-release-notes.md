# v0.2.0 Release Notes

## Summary

`v0.2.0` turns `ai-engineering-harness` into a markdown-first Harness Design System focused on project-specific harness profiles, selected workflows, selected team patterns, memory rules, quality gates, and durable goal artifacts.

## What Is Included

- Harness Design System target
- `harness-build` command
- harness profile templates
- team architecture selection
- memory model and memory safety
- SDLC execution model
- skill authoring system
- demo harness build
- harness build contract validation
- `harness-build` usage guides
- target repository validation design
- `v0.2.0` readiness and scope docs

## What Changed From v0.1.0

Compared with `v0.1.0`, this release adds the missing project-specific design surfaces that move the repository beyond a generic markdown operating kit:

- a documented harness builder flow
- profile templates for harness, team, skills, workflow, gates, and memory
- explicit team pattern selection guidance
- explicit memory and SDLC operating models
- skill authoring and selection guidance
- a full demo harness build example
- stronger lightweight validation for the new design-system artifacts
- operator-facing docs for using and reviewing `harness-build`

## What Is Intentionally Not Included

- full target repo validator implementation
- runtime adapters
- server, database, or background systems
- package publishing automation
- large skill catalog import

## Who Should Use This Release

Use `v0.2.0` if you want:

- a markdown-first harness for AI-assisted engineering work
- project-specific `.harness/` profile artifacts before implementation
- explicit workflow, review, memory, and quality-gate guidance
- a runtime-light operating model that can be adopted into a host repository

## How To Validate

Run:

```bash
node bin/validate.js
npm test
```

## How To Adopt

1. Read [AGENTS.md](../AGENTS.md).
2. Use [docs/adoption-guide.md](adoption-guide.md).
3. Run `harness-build` using [docs/harness-build-usage.md](harness-build-usage.md).
4. Review the resulting profile with [docs/harness-build-review-checklist.md](harness-build-review-checklist.md).

## Known Limitations

- host repository target validation is documented but not implemented
- validation remains structural and lightweight rather than semantic
- the system is still intentionally manual and markdown-first
- no runtime coordination layer is provided

## Next Likely Milestone

The next likely milestone is `v0.3.0`, focused on optional lightweight enhancements such as host-repository validation implementation, while keeping heavy runtime systems out of scope by default.
