# v0.3.0 Readiness

## Purpose

This document records whether `v0.3.0` is ready to release as Lightweight Host Repository Validation.

The goal of this pass is to audit scope completion, validation evidence, and safety boundaries without expanding the feature set.

## Target Summary

`v0.3.0` adds lightweight validation support for adopted host repositories that use `.harness/` artifacts.

The release keeps validation dependency-free, structural-only, and scoped to expected harness paths.

## Completed Capabilities

- target profile validation for adopted host repositories
- target goal validation for adopted host repositories
- CLI argument parsing for `--target`, `--profile-only`, and `--goal`
- valid and invalid target fixtures for profile and goal validation
- target validation usage documentation
- explicit `--profile-only` mode
- structural-only validation boundary
- reuse of existing heading contracts where practical

## Remaining Gaps

- no semantic validation of application behavior
- no secret scanning
- no runtime adapters
- no deep repository scanning
- no automated harness generation

These gaps are intentional and remain out of scope for `v0.3.0`.

## Release Blockers

No release blockers are currently identified.

The implemented `v0.3.0` scope is present, documented, and verified by the existing validation and test flows.

## Non-Blockers

- lack of semantic validation
- lack of secret detection
- lack of runtime-specific adapters
- lack of deep repository analysis
- lack of automated harness generation
- lack of optional context warnings

These are non-blockers because `v0.3.0` is explicitly defined as lightweight structural validation rather than a broader repository analysis system.

## Validation Status

- `node bin/validate.js` passes for this repository
- target validation modes are implemented in `validate.js`
- required v0.3.0 docs are part of the repository validation contract

## Test Status

- `npm test` passes
- tests cover default repository validation
- tests cover target profile validation
- tests cover target goal validation
- tests cover valid and invalid fixtures
- tests cover CLI usage errors for invalid flag combinations

## Safety Review

- no dependencies were added
- no `src/` directory was added
- no markdown parser was added
- no runtime adapters were added
- no server, database, Docker, LangGraph, or background workers were added
- validation remains structural and contract-focused
- validation does not scan arbitrary repository files
- validation does not claim to prove application correctness

## v0.3.0 Release Decision

`v0.3.0` is ready to release.

The implemented scope matches the stated target: lightweight structural validation for this harness repository, adopted host repository profiles, and adopted host repository goal artifacts.
