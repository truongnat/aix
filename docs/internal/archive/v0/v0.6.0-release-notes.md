# v0.6.0 Release Notes

## Summary

`v0.6.0` is a docs-only Runtime Consumption Guides release.

It explains how common AI coding runtimes should consume the plugin-like markdown capability pack inside a target repository while keeping the harness runtime-light and markdown-first.

## What Is Included

- runtime consumption model
- runtime docs index update
- Claude Code capability pack consumption guide
- Cursor capability pack consumption guide
- Codex capability pack consumption guide
- Gemini CLI capability pack consumption guide
- OpenCode capability pack consumption guide
- runtime comparison guide
- `v0.6.0` readiness and release scope docs
- shared prompt and validation patterns

## What Changed From v0.5.0

`v0.5.0` clarified the pack as a plugin-like capability with source-repo versus target-repo boundaries and distribution modes.

`v0.6.0` adds runtime-facing usage guidance on top of that model:

- a shared runtime consumption model for target-repo operation
- per-runtime guides reframed around install-or-copy, read-first, profile, goal, and structural validation
- a comparison guide to choose a runtime and reuse common prompt and validation patterns
- explicit safety boundaries against adapters, integrations, and treating the source pack as the product repo

## What Is Intentionally Not Included

- runtime adapters
- runtime integrations
- compiled or installed plugins
- marketplace package
- package publishing automation
- server, database, or background systems
- semantic validation
- deep repository scanning

## Who Should Use This Release

Use `v0.6.0` if you:

- already consume the harness as a capability pack in a target repository
- work in Claude Code, Cursor, Codex, Gemini CLI, or OpenCode
- want runtime-specific read-first and validation guidance without new code or dependencies
- need a comparison guide to pick the best runtime guide for your workflow

## How To Use Runtime Consumption Guides

1. Read [Runtime Consumption Model](runtime-consumption-model.md) and [Consume As Pack](consume-as-pack.md).
2. Install or copy the pack into your target repository.
3. Open [Runtime Guides](runtimes/README.md) or start with [Runtime Comparison](runtimes/comparison.md) if unsure which runtime to use.
4. Follow the guide for your runtime: read `AGENTS.md` and installed docs first, then `.harness/` artifacts.
5. Validate structurally in the target repo:

```bash
node bin/validate.js --target <path> --profile-only
node bin/validate.js --target <path> --goal <goal-id>
```

Treat validation as structural only. It does not prove application correctness.

## Known Limitations

- no runtime adapters or integrations exist
- no marketplace or package publishing automation exists
- runtime docs may need updates if tools change
- validation remains structural-only
- guides assume you already understand source pack vs target repo from `v0.5.0`

The next likely milestone is optional follow-up work: runtime doc maintenance as tools evolve, optional runtime adapters, or optional marketplace or packaging automation—only if the project explicitly expands beyond docs-only runtime guidance.
