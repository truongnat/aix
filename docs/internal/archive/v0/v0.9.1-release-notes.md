# v0.9.1 Release Notes

## Summary

`v0.9.1` is the **Experimental Runtime-Native Installer** release.

It ships runtime- and scope-aware installation (`install.sh` + `install-runtime.js`), project-local `.harness/` init, runtime-aware target validation, and dogfood evidence for Scenarios **C** and **D1–D6**. This is **not** `v1.0.0` and does **not** claim stable runtime-native support.

## Release Type

- **Pack version:** `v0.9.1`
- **Label:** experimental runtime-native installer
- **Prior tag:** `v0.9.0` (stable contract freeze documentation)
- **Next likely work:** manual runtime verification sprint or `v0.10.x` stable-runtime narrowing — not immediate `v1.0.0`

## What Is Included

- Runtime selector (`--runtime`) and scope selector (`--scope`) in [install.sh](../install.sh)
- [install-runtime.js](../lib/install-runtime.ts) and [runtime/](../runtime/) payloads
- Project `.harness/` init (`--scope project --init-harness`) with dry-run, skip, and `--force`
- Runtime-aware validation: `node bin/validate.js --target <repo> --runtime <name> --profile-only`
- Dogfood reports: Scenario C, D1–D6
- [runtime-dogfood-summary.md](runtime-dogfood-summary.md), [v0.9.x-readiness.md](v0.9.x-readiness.md), [v0.9.x-release-scope.md](v0.9.x-release-scope.md)
- D2 patch: `.harness/` init no longer creates `AGENTS.md`; `generic`/`codex` runtime writes `AGENTS.project.md`

## Dogfood Evidence

Consolidated in [runtime-dogfood-summary.md](runtime-dogfood-summary.md):

- **File/install:** PASS for D1–D6 (runtime-native paths; no root pack pollution)
- **Runtime-aware validation:** PASS where exercised
- **Manual runtime checks:** mostly not run, blocked (Codex quota), or inconclusive (Gemini `extensions list`)
- **Stable claim:** **No** for every runtime-native mode

## Validation

From the source pack:

```bash
node bin/validate.js
npm test
node bin/validate.js --target test/fixtures/valid-target-profile --profile-only
node bin/validate.js --target test/fixtures/valid-target-goal --goal google-login
node bin/validate.js --target test/fixtures/valid-target-profile-cursor --runtime cursor --profile-only
```

After install into a target repo, use `--runtime` matching the install (see [runtime-aware-validation.md](runtime-aware-validation.md)).

## What Is Experimental

- All runtime-native modes in the table above — evidence is **structural** (files created, validator passes), not proven in live IDE/CLI sessions
- Project-local Gemini extension load (best-effort)
- Global installs when dry-run shows **UPDATE** on existing user config
- Combined `--runtime all`

## What Is Intentionally Not Included

- Stable runtime-native support claims
- npm publishing or package registry distribution
- Automatic Claude Code marketplace plugin install
- Semantic validation (content quality, plugin load detection)
- Verified manual sessions for every runtime on this tag
- Root-copy as recommended default UX
- `v1.0.0` contract re-freeze for runtime-native as final behavior

## Known Limitations

- Legacy `node bin/validate.js --target <repo> --profile-only` requires `AGENTS.md` — fails on cursor/opencode/gemini/claude-only repos; use `--runtime`
- Codex manual check blocked by API quota during dogfood
- Gemini `gemini extensions list` did not confirm extension load
- Claude `/plugin install` and global settings merge not automated; global write skipped when existing `~/.claude/settings.json` would UPDATE
- `package.json` version is packaging metadata; pack authority is `PACK.md`

## Upgrade / Adoption Notes

**Recommended** (preview then apply):

```bash
sh install.sh --runtime cursor --scope project --init-harness --dry-run --yes
sh install.sh --runtime cursor --scope project --init-harness --yes
```

Remote equivalent:

```bash
curl -fsSL https://raw.githubusercontent.com/truongnat/ai-engineering-harness/main/install.sh | sh -s -- \
  --ref v0.9.1 --runtime cursor --scope project --init-harness --dry-run --yes
```

- Use `--init-harness` for project `.harness/` scaffold
- Validate from source pack with matching `--runtime`
- **Manual fallback** (`--runtime manual` or default non-interactive `curl | sh`) remains available but is **not** the recommended final UX
- Pin `--ref v0.9.1` for reproducible installs ([plugin-install-security.md](plugin-install-security.md))

From `v0.9.0` contract-freeze tag: no breaking change to frozen contract **documents**; install UX adds new paths alongside legacy `install.js` root copy.
