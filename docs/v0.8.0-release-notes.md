# v0.8.0 Release Notes

## Summary

`v0.8.0` is a Real Capability Pack Dogfood release.

It proves the plugin-like markdown capability pack works when installed into external target repositories, with harness profile and goal artifacts validated structurally, and records evidence toward `v0.9.0` contract freeze and `v1.0.0`.

## What Is Included

- real capability pack dogfood plan
- dogfood scenarios
- dogfood report template
- friction log
- follow-up backlog
- Scenario A tiny repo health-check dogfood report
- Scenario B frontend/mobile feature dogfood report
- dogfood fix pass
- harness example to target layout guide
- `v0.8.0` readiness and release scope docs
- install.js `targetDisplay` for clearer install summary paths

## What Changed From v0.7.0

`v0.7.0` defined packaging contracts (`PACK.md`, manual packaging, verification checklist).

`v0.8.0` adds real-world consumption evidence:

- two executed dogfood scenarios in distinct external target repos
- documented friction and a classified follow-up backlog
- doc fixes from dogfood (source-pack validation flow, example-to-target layout)
- confirmation that install → profile → goal → `--target` validation works outside the source pack

## Dogfood Evidence

### Scenario A

- tiny local Node HTTP target repo
- runtime: Cursor
- goal: `health-check`
- profile and goal structural validation: **pass**

### Scenario B

- semi-real Flutter-style mobile skeleton (distinct repo)
- runtime: Codex
- goal: `google-login` (guest mode preserved in planning artifacts)
- profile and goal structural validation: **pass**

### Minimum bar

- **two distinct target repositories**
- **two runtime patterns** (Cursor, Codex)
- **no v1 blockers** identified in follow-up backlog

Reports: [scenario-a-tiny-health-check.md](pack-dogfood-reports/scenario-a-tiny-health-check.md), [scenario-b-frontend-mobile-feature.md](pack-dogfood-reports/scenario-b-frontend-mobile-feature.md).

## What Is Intentionally Not Included

- Scenario C as a release requirement
- runtime adapters
- package publishing automation
- marketplace automation
- archive generation automation
- semantic validation
- minimal install tier implementation
- server, database, or background systems

## Who Should Use This Release

Use `v0.8.0` if you:

- need evidence that the pack works in real target repos, not only in source-pack self-validation
- are preparing for `v0.9.0` contract freeze or `v1.0.0`
- want dogfood reports, friction log, and layout guidance before adopting in mobile or tiny repos
- already use `v0.5.0`–`v0.7.0` distribution and packaging docs

## Known Limitations

- dogfood did not implement application features (planning and structural validation only)
- default install surface remains large (~83 paths) for tiny repos
- no Scenario C execution in this release
- validation remains structural-only
- external dogfood target repos are not part of this repository

## Next Likely Milestone

`v0.9.0` Stable Contract Freeze — lock `PACK.md`, installed surface, harness profile, goal artifacts, validation behavior, and packaging flow before `v1.0.0`.
