# v0.9.2 Release Notes

## Summary

`v0.9.2` is the **experimental simple lifecycle CLI + capability cache + git hygiene** release.

It completes the minimum project runtime-native lifecycle:

- `install`
- `update`
- `uninstall`
- `status`
- `doctor`

while keeping runtime support explicitly **experimental**.

## Release Type

- Experimental runtime-native installer polish
- Documentation and UX hardening release
- Not a stable runtime support release

## What Changed Since v0.9.1

- `aih.sh` is now the preferred lifecycle dispatcher
- `install.sh` is now a compatibility wrapper
- simple command defaults exist for install/update/uninstall
- project runtime-native installs use `.ai-harness/` as capability source
- private mode uses `.git/info/exclude` instead of `.gitignore`
- project uninstall and update are implemented
- `status` and `doctor` are available
- simple lifecycle dogfood exists for the new command model

## New CLI Entrypoint

Preferred entrypoint:

```bash
sh aih.sh install
```

Remote:

```bash
curl -fsSL https://raw.githubusercontent.com/truongnat/ai-engineering-harness/main/aih.sh | sh -s -- install
```

`install.sh` remains available as a compatibility wrapper and future installer surface, but it is no longer the preferred lifecycle command name.

## Simple Commands

```bash
sh aih.sh install
sh aih.sh update
sh aih.sh uninstall
sh aih.sh uninstall --all
sh aih.sh status
sh aih.sh doctor
```

## Capability Cache

Project runtime-native installs now follow:

```txt
provider entrypoint
    ->
.ai-harness/   capability source
    ->
.harness/      project state
```

`.ai-harness/` contains pack capabilities such as `AGENTS.md`, `commands/`, `skills/`, `workflows/`, `patterns/`, `templates/`, and selected docs.

`.harness/` remains project-specific memory, profile, and goals.

## Git Hygiene

Private project installs use `.git/info/exclude`, not `.gitignore`.

That keeps generated harness/runtime files off normal `git status` output without editing tracked repo files.

## Install Lifecycle

Default `install` behavior in a detectable project:

- scope: `project`
- visibility: `private`
- ignore strategy: `info-exclude`
- install `.ai-harness/`
- init `.harness/` when missing
- auto-detect runtime from provider hints when possible

If provider detection fails in non-interactive mode, pass `--runtime cursor|claude|gemini|opencode|generic|codex`.

## Update Lifecycle

`update` refreshes:

- `.ai-harness/`
- selected runtime entrypoint

It preserves `.harness/`.

## Uninstall Lifecycle

Default `uninstall` removes only runtime entrypoints and cleans the local exclude block.

It keeps:

- `.ai-harness/`
- `.harness/`

Use `uninstall --all` for full cleanup.

## Status / Doctor

`status` reports local install state:

- target
- Git repo detection
- detected runtimes
- `.ai-harness` present or missing
- `.harness` present or missing
- exclude block present or missing

`doctor` verifies local readiness without network.

## What Remains Experimental

- Stable runtime support is **not** claimed
- Manual runtime/IDE session proof is still pending
- `--runtime all` is still not a stable promoted mode
- Global update remains planned-only
- Antigravity is still not implemented

## Known Limitations

- Non-interactive install without provider hints still requires explicit `--runtime`
- Default uninstall removes the local exclude block even when `.ai-harness/` and `.harness/` are kept, so private repos may become noisy again until full uninstall
- Manual fallback still exists but is not the recommended main UX

## Migration Notes

From `v0.9.1` to `v0.9.2`:

- prefer `aih.sh` over `install.sh` for lifecycle commands
- continue using `install.sh` only when compatibility matters
- use runtime-aware validation for runtime-native targets
- expect `.ai-harness/` to exist for project runtime-native installs

## Next Likely Milestone

Likely next step after `v0.9.2`:

- `v0.10.0` binary `aih` installer surface
- manual runtime verification sprint
- no stable runtime claim until those checks pass
