# v0.11.0 — Scope freeze, contracts, dogfood, and CI trust

**Status:** experimental — feature scope frozen for this release. No further v0.11.0 feature work planned.

## Summary

v0.11.0 narrows provider scope, clarifies product identity, adds markdown execution contracts with stronger validation, ships a tiny end-to-end dogfood example, and exposes CI + public demo polish for external trust.

---

## Step 1 — Provider scope and identity

### Provider scope

- **Active:** Claude Code (primary), Cursor (secondary), Codex (experimental), Gemini (experimental).
- **Removed from active scope:** OpenCode — no wizard, `install-runtime`, or matrix install path; legacy cleanup via `aih.sh uninstall --runtime opencode` only.
- Historical CHANGELOG and dogfood reports may still mention OpenCode.

### Product identity

- Public positioning: **lightweight, markdown-first workflow kit** for AI coding agents.
- Not positioned as a framework, harness OS, orchestrator, or universal plugin runtime.
- README: **Why not a framework?**, **Known limitations**, **Release discipline**.

---

## Step 2 — Command and skill contracts

- Harness commands: minimum read sets, preconditions, required outputs, redirect behavior, failure conditions, completion gates.
- Skills: `When Not To Use`, `Inputs`, `Output Contract`, `Common Failure Modes`.
- `VERIFY.md` evidence-first: `status`, tests run, manual checks, evidence, known gaps.
- `PLAN.md` **Approval Status** (`status: draft | approved | blocked`) for `harness-run` gating.
- [AGENTS.md](../AGENTS.md): progressive loading and phase discipline.

---

## Step 2.1 — Contract validation (structure, not runtime)

- `validate.js` checks substantive section bodies (not heading-only placeholders).
- Command redirects must mention `harness-*` commands; failure conditions need concrete negative rules.
- Templates: VERIFY `status:` field; PLAN approval fields; dogfood demo contracts.

## Known limitations (honest)

- **Dogfood type:** documents the harness workflow with checked-in artifacts; does not prove every provider-native install path in the example folder.
- **Providers:** Claude and Cursor are the most exercised paths; Codex and Gemini remain experimental; no stable per-provider guarantee.
- **Installer:** `aih.sh` expects POSIX `sh`; Windows users should prefer `npx ai-engineering-harness install` (Git Bash / WSL for shell fallback).
- **Enforcement:** contracts are validated markdown structure, not a long-running runtime or telemetry layer.
- `runtime/opencode/` may remain in the repo for reference; it is not an active install target.
