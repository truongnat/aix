# v0.5.0 Readiness

## Purpose

Audit whether `v0.5.0` is ready to release as a plugin-distribution-model clarification release.

## Target Summary

`v0.5.0` pivots the project from clone-repo thinking toward a plugin-like markdown capability pack.

The release clarifies how the harness should be consumed, what belongs in the installed surface, and how future distribution modes should be understood without adding automation.

## Completed Capabilities

- plugin-like positioning in `TARGET.md`
- plugin model docs
- distribution model docs
- installed surface contract
- consumption mode guide
- consume-as-pack quick guide
- release archive model
- release archive checklist
- clearer source repo vs target repo distinction

## Remaining Gaps

- no package or marketplace automation
- no runtime adapters
- no archive generation script
- no global agent capability folder integration
- no registry publishing

## Release Blockers

No release blockers identified.

The intended `v0.5.0` scope is documentation and distribution-model clarification, and that surface is now present.

## Non-Blockers

These are intentional non-blockers for `v0.5.0`:

- no package or marketplace automation
- no runtime adapters
- no archive generation script
- no global agent capability folder integration
- no registry publishing

These are outside the release target because `v0.5.0` clarifies the distribution model rather than implementing automation.

## Validation Status

Current validation status:

- `node validate.js` passes
- target profile fixture validation passes
- target goal fixture validation passes

Validation remains structural-only and no new runtime behavior was introduced.

## Test Status

Current test status:

- `npm test` passes
- install surface tests remain in place
- target profile and target goal validation coverage remains in place

## Safety Review

- markdown remains the source of truth
- no dependencies were added
- no runtime adapters were added
- no marketplace or package-publishing automation was added
- no archive generation was added
- no semantic validation or deep scanning was added

## v0.5.0 Release Decision

`v0.5.0` is ready to release as a plugin-distribution-model clarification release.

It makes the pack-consumption story explicit without expanding into packaging automation or runtime features.
