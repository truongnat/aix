# v1.2.2 Release Notes

## Codex CLI slash command support

`/harness-*` slash commands now work in Codex CLI via **UserPromptSubmit hook routing**.

When a user types `/harness-plan` (or any `/harness-*` command) in Codex CLI, the hook in `.codex/hooks/core/codex-hook-router.js` intercepts the prompt, reads the matching command file from `.codex/commands/`, and injects it as `additionalContext`. This gives functional parity with Claude Code and Cursor.

### Available commands

- `/harness-start` — Session Start: restore context, route session, map repository
- `/harness-discuss` — Discuss scope, constraints, and approach before planning
- `/harness-plan` — Translate goal into an explicit, reviewable implementation plan
- `/harness-run` — Execute the approved plan
- `/harness-verify` — Verify implementation against acceptance criteria
- `/harness-ship` — Ship the verified change
- `/harness-remember` — Record session learnings
- `/harness-map` — Manual context refresh
- `/harness-status` — Show current harness state
- `/harness-doctor` — Diagnose harness installation issues

### Install

```bash
npx ai-engineering-harness install --provider codex --target .
```

The installer now copies `commands/harness-*.md` to `.codex/commands/` and sets up the hook router that intercepts `/harness-*` prompts.

## Bug fix: .harness/ git ignore

`.harness/` is now always added to `.git/info/exclude` during project install, regardless of whether it already exists. Previously, `.harness/` was only ignored on first install (when `initHarness` was true); re-installing would skip the ignore entry.

## Workflow

The core harness workflow loop is unchanged:

Session Start → Discuss → Plan → Run → Verify → Ship → Remember

## Files changed

- `hooks/core/codex-hook-router.js` — slash command interception via `resolveSlashCommand()`
- `lib/install-runtime.ts` — new `installCodexCommands()` copies command files to `.codex/commands/`
- `lib/backend/constants.ts` — `.harness/` always in ignore list; excluded from uninstall paths
- `lib/catalog/provider-command-metadata.ts` — Codex declared as `nativeSlashCommands: true`
- `providers/codex.json` — updated status and install paths
- `rules/providers/codex/AGENTS.md` — lists available slash commands
- `docs/codex-plugin-support.md` — updated documentation
