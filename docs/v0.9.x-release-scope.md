# v0.9.x Release Scope

## Purpose

Define what is **in** and **out** of an experimental **v0.9.x** release after runtime-native installer work and D1–D6 dogfood.

**Release vehicles:**

- **`v0.9.1`** — Experimental Runtime-Native Installer ([v0.9.1-release-notes.md](v0.9.1-release-notes.md)) — shipped
- **`v0.9.2`** — Installer UX + Git Hygiene ([installer-ux-v0.9.2-plan.md](installer-ux-v0.9.2-plan.md)) — design/implementation in progress

Distinct from [v0.9.0-release-scope.md](v0.9.0-release-scope.md) (contract freeze) and [v0.9.x-readiness.md](v0.9.x-readiness.md) (release decision).

## In Scope

- Runtime selector (`--runtime`) and scope selector (`--scope`) in [install.sh](../install.sh)
- [install-runtime.js](../install-runtime.js) project/global writes from [runtime/](../runtime/) payloads
- Project `.harness/` init (`--scope project --init-harness`) with dry-run, skip, `--force`
- Rejection of global `.harness/` init
- Runtime-aware target validation (`validate.js --target --runtime <name>`)
- Runtime-native **project** installs dogfooded: `generic`, `codex`, `cursor`, `opencode`, `gemini`, `claude`
- Dogfood reports: Scenario C, D1–D6
- [runtime-dogfood-summary.md](runtime-dogfood-summary.md), [runtime-native-install-audit.md](runtime-native-install-audit.md), [runtime-aware-validation.md](runtime-aware-validation.md)
- Experimental support table in [plugin-install-ux.md](plugin-install-ux.md)
- Manual fallback (`--runtime manual`, legacy `install.js` root copy) retained and dogfooded (C)
- `windsurf` as cursor alias (documented, not separately dogfooded)
- Automated tests for install, init, runtime payloads, runtime-aware validation

## Out of Scope

- **Stable** runtime-native support claims
- npm publishing or package registry distribution
- Automatic Claude Code marketplace plugin install
- Verified manual sessions for every runtime in this release
- Semantic validation (content quality, plugin load state)
- Root-copy as **recommended** default UX
- `--runtime all` stable or dogfooded support
- v1.0.0 contract re-freeze for runtime-native as final behavior
- Marketplace automation, checksums, global CLI (`ai-harness`) unless separately shipped
- Mandatory global writes on adopters' machines without dry-run consent

## Experimental Labeling

Ship with explicit **experimental** wording for:

- All runtime-native modes in the D1–D6 matrix
- Project-local Gemini extension path (best-effort)
- Combined `all` runtime

## Success Criteria For This Release

- Adopter can install one runtime + project `.harness/` without root pack pollution
- Adopter can validate with matching `--runtime`
- Maintainer can trace evidence in dogfood reports and summary doc
- `node validate.js` and `npm test` pass on release branch
