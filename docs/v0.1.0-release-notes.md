# v0.1.0 Release Notes

## Summary

`v0.1.0` is the first manual release of `ai-engineering-harness` as a markdown-first operating model for engineering agents.

## What Is Included

- `AGENTS.md` operating contract
- command loop documentation
- core skills
- workflows and patterns
- artifact templates
- adoption and runtime usage docs
- skill packs and workflow scenarios
- quality gates and release-readiness docs
- lightweight install and validation helpers

## What Is Intentionally Not Included

- runtime adapters
- release automation
- package publishing automation
- server code
- Docker
- LangGraph
- databases or background workers

## Who Should Use This Release

- teams experimenting with disciplined AI-assisted engineering workflows
- maintainers who want explicit markdown artifacts for planning, verification, and memory
- repositories that want a lightweight adoption path without adding platform complexity

## How To Validate

Run:

```bash
node validate.js
```

## How To Adopt

Start with:

- `docs/adoption-guide.md`
- `docs/adoption-smoke-test.md`
- `docs/runtime-compatibility.md`

## Known Limitations

- the operating model is still stabilizing under `v0.x`
- adoption still depends on humans and agents keeping `.harness/` artifacts current
- validation is structural and lightweight rather than exhaustive

## Next Likely Milestone

`v0.2.0`: richer validation and adoption polish while staying markdown-first and runtime-light.
