# Frozen Packaging And Release Contract

## Purpose

Record manual packaging and release behavior frozen for `v1.0.0` as part of `v0.9.0` Stable Contract Freeze.

## Contract Status

**Frozen for v1.0.0** (v0.9.0 Step 5).

## Manual Packaging Flow

v1.0.0 packaging is **manual**:

1. confirm release scope and non-goals
2. update [PACK.md](../PACK.md) **Pack Version** and body alignment
3. complete [pack-verification-checklist.md](pack-verification-checklist.md)
4. run validation commands from source pack root
5. tag source repository (maintainer choice)
6. assemble release archive per [release-archive-checklist.md](release-archive-checklist.md) if distributing an archive

See [manual-packaging-guide.md](manual-packaging-guide.md) for step-by-step maintainer guidance.

## Release Archive Expectations

- release archive shape is **documented**, not generated by automation in v1.0.0
- archive is a deliberate subset aligned with [release-archive-model.md](release-archive-model.md)
- archive is not a full git mirror of maintenance-only docs unless explicitly included

## PACK.md In Archives

- [frozen PACK.md contract](frozen-pack-contract.md): `PACK.md` at **release archive root**
- adopters use it for pack identity and validation command reference
- target repos do **not** require local `PACK.md` by default

## Verification Checklist

- [pack-verification-checklist.md](pack-verification-checklist.md) is the manual gate before tag/archive
- checks manifest, installed surface alignment, runtime docs, and validation commands
- structural `node validate.js` does not replace the checklist for body alignment

## Release Checklist

- [release-checklist.md](release-checklist.md) and [release-archive-checklist.md](release-archive-checklist.md) integrate versioning and archive steps
- release notes and CHANGELOG updates are manual markdown

## What Is Guaranteed

- `PACK.md` travels with release archives at archive root
- [frozen-pack-contract.md](frozen-pack-contract.md) headings validated in source pack
- manual packaging guide and verification checklist remain the v1 process
- no requirement for npm or marketplace distribution in v1.0.0

## What Is Not Guaranteed

- archive zip/tar **generation scripts**
- npm package publishing or marketplace automation
- manifest body semantic validation or checksums
- automated promotion between environments

## Allowed Additive Changes

- checklist items if additive and non-breaking
- release notes and scope docs for each version
- clearer manual archive assembly steps

## Breaking Changes

See [breaking-change-policy.md](breaking-change-policy.md).

Breaking for this contract:

- removing `PACK.md` from release archive expectations
- requiring archive generation scripts or npm publish for v1 adoption
- replacing manual verification with undocumented automation requirements
