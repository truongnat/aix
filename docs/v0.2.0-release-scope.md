# v0.2.0 Release Scope

This document defines exactly what is in scope and out of scope for `v0.2.0`.

## In Scope

`v0.2.0` includes:

- `harness-build` command
- project-specific harness profile artifacts
- team architecture selection
- memory model and memory safety
- SDLC execution model
- skill authoring system
- demo harness build
- harness-build usage docs
- target repo validation design

## Out Of Scope

`v0.2.0` does not include:

- full target repo validator implementation
- runtime adapters
- server, database, or background systems
- package publishing automation
- large skill catalog import
- external memory backend

## Scope Boundary Notes

- target repository validation is documented for future implementation, but not implemented in this release
- `validate.js` remains a lightweight repository validator for this harness repository itself
- markdown remains the source of truth
- release success is measured by design-system completeness, documentation clarity, and lightweight validation rather than runtime expansion
