# v0.3.1 Release Notes

## Summary

`v0.3.1` is a small documentation patch release after the `v0.3.0` dogfood pass.

It records local target-repository dogfood results and captures compact `v0.4.0` direction notes without expanding the validator feature set.

## What Is Included

- dogfood notes for local target repository validation
- `v0.4.0` idea notes based on dogfood findings
- release metadata updates for the patch release

## What Changed From v0.3.0

`v0.3.0` shipped Lightweight Host Repository Validation.

`v0.3.1` adds:

- written evidence that installer dry-run and real install worked on a toy local target repository
- written evidence that target profile validation passed
- written evidence that target goal validation passed
- written evidence that intentionally broken profile and goal artifacts were caught clearly
- compact next-milestone notes for ergonomics-focused follow-up work

## Dogfood Result

The dogfood run confirmed:

- install dry-run worked
- real install worked
- target profile validation passed
- target goal validation passed
- intentional profile break was caught
- intentional goal break was caught

The run found ergonomics opportunities, but no release-blocking issues.

## What Is Intentionally Not Included

- new validation features
- semantic application correctness checks
- application code validation
- runtime adapters
- server, database, or background systems
- markdown parser
- deep repository scanning

## Known Limitations

- validation remains structural-only
- validation does not prove application correctness
- installer output is still accurate but visually dense on first use
- the release does not add automation or runtime systems

## Next Likely Milestone

The next likely milestone is `v0.4.0` ergonomics polish around install-to-profile flow, validation clarity, and richer adoption examples, while staying markdown-first and runtime-light.
