# v1.2.0 - Production-Readiness Hardening

**Status:** minor release. The core workflow loop and provider support posture are unchanged.

## Highlights

- **Zero `any` types** - all 12 untyped values in `lib/` replaced with proper interfaces and type assertions for strict TypeScript safety.
- **Hardened JSON parsing** - 12+ unguarded `JSON.parse` calls now wrapped in try/catch with safe fallbacks across evals, policy engine, insights, and provider registry.
- **Security improvements** - shell injection pattern fix in evals, info disclosure removal from telemetry response, regex cache size limit with LRU eviction in policy engine.
- **Test coverage boost** - 84 new tests (362 → 446), covering CLI args, policy engine, harness config, validate/target, and cli-ui.
- **CI hardening** - added typecheck and security audit steps, upgraded GitHub Actions dependencies.

## What changed

### Added

- `InstallManifest` class for atomic install with rollback on partial failure.
- `--skip-demo-eval` CLI flag support (was advertised but threw "Unknown argument").
- 84 new tests across 5 new test files: `cli-args`, `policy-engine`, `harness-config`, `validate-target`, `cli-ui`.
- TypeScript strict typecheck and `npm audit` steps in CI workflow.
- `SECURITY.md` with vulnerability disclosure policy and 30-day timeline.

### Changed

- All `any` types in `lib/` eliminated (12 → 0) with proper interfaces (`LlmJudgeResponse`, `Check`, `CheckResult`, typed callbacks).
- All unguarded `JSON.parse` calls hardened with try/catch: `evals/index.ts`, `evals/mode-mutations.ts`, `evals/task-registry.ts`, `insights/harness-config.ts`, `policy/engine.ts`, `policy/generator.ts`, `provider-registry.ts`.
- `FORBIDDEN_SHELL_PATTERN` in evals now includes `%` character.
- Policy engine regex cache limited to 1000 entries with LRU eviction and pattern length validation (max 1000 chars).
- `softprops/action-gh-release` upgraded from v1 to v2 in publish workflow.
- Plugin versions synced to 1.2.0 across `.claude-plugin` and `.cursor-plugin`.
- Site version synced to 1.2.0.

### Fixed

- Cursor project-scope install was silently a no-op; now correctly writes `.cursor/rules/` in project scope.
- `void packRoot` suppression in install-runtime was hiding a variable actually used downstream.
- Telemetry server 202 response no longer leaks `storagePath`.
- `bin/` scripts now have executable permission bits set.

## The loop is unchanged

```text
Session Start → Discuss → Plan → Run → Verify → Ship → Remember
```

## Verification

- `npm run build`
- `npm test`
- `node bin/validate.js`

## Upgrade notes

- No breaking changes. All fixes are backwards-compatible.
- Existing `.harness/` and `.ai-harness/` install surfaces require no migration.
- The Cursor install fix only affects new project-scope installs; existing installs are unaffected.
