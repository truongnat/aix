# Product Management Pro

Expert-level orchestration of the product lifecycle. Focuses on bridging user needs, business goals, and technical feasibility.

## Boundary

**`product-management-pro`** covers Product Discovery, PRD (Product Requirements Document) creation, Roadmapping (Gantt, Kanban), User Research, and Stakeholder Management. It coordinates with `ux-design-pro` for design and `senior-architect` for technical feasibility.

## When to use

- Defining the vision and roadmap for a new product or feature.
- Writing a comprehensive PRD to align engineering and design teams.
- Conducting user interviews and synthesising feedback into actionable items.
- Prioritizing backlogs using frameworks like RICE or MoSCoW.

## Workflow

1. **Discovery**: Identify user problems through research and data analysis.
2. **Definition**: Define the MVP, value proposition, and success metrics (KPIs).
3. **PRD Writing**: Document requirements, user stories, and acceptance criteria.
4. **Roadmapping**: Plan the sequence of delivery across time.
5. **Execution**: Lead Agile rituals and manage the product backlog.
6. **Iteration**: Analyze post-launch data and user feedback for improvements.

### Operating principles

- **Problem First, Solution Second**: Deeply understand the "Why" before deciding the "What".
- **Data-Informed**: Use metrics and user research to drive decisions.
- **Radical Transparency**: Keep all stakeholders aligned through clear documentation.
- **Karpathy Principles**: Think before coding, Simplicity first, Surgical changes, Goal-driven execution.

## Quick example

**Vision:** "Build a collaborative code-review tool that feels as fast as real-time chat."

**Success Metrics:** 
- Average time to first comment.
- User satisfaction (CSAT) score.
- PR merge velocity.

## Checklist before calling the skill done

- [ ] **Think Before Coding**: User problem and market fit analyzed before defining features.
- [ ] **Simplicity First**: Minimalist MVP defined to validate core hypotheses.
- [ ] **Surgical Changes**: Feature scope kept tight to ensure rapid delivery.
- [ ] **Goal-Driven Execution**: Success metrics (KPIs) defined and measurable.
- [ ] PRD includes clear acceptance criteria and edge cases.
- [ ] Roadmap is realistic and aligned with technical constraints.
- [ ] User research/feedback loops integrated into the process.
