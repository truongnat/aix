---

# 2️⃣ WORKFLOW MODEL (Domain-driven)

Workflow không phải chỉ là list steps.

Nó phải là:

```yaml
meta:
  domain: backend
  roles: ["architect", "reviewer"]
  rules: ["strict-types", "clean-arch"]
  skills: ["analyze_code", "generate_tests"]

steps:
  - id: analyze
    skill: analyze_code
    template: analyze_backend.tpl

  - id: implement
    skill: implement_feature
    depends_on: analyze

  - id: validate
    skill: run_script
    script: "cargo test"
```

👉 Workflow = Declarative execution graph.

---

# 3️⃣ SKILL SYSTEM (Pluggable + Cross-call)

Skill phải có:

```
SkillMeta:
 - name
 - domain
 - roles
 - rules
 - can_call: [other skills]
```

Skill execution:

```
Skill.execute(context, memory, config)
```

Skill có thể:

* gọi skill khác
* gọi script ngoài
* gọi Ollama sub-agent
* query vector DB

---

# 4️⃣ SUB-AGENT MODEL (Ollama Local)

Mỗi skill AI-heavy không gọi trực tiếp model.

Nó gọi:

```
SubAgent {
    role: "architect",
    model: "deepseek-coder",
    temperature: 0.2,
    system_template: ...
}
```

SubAgent build prompt theo:

```
role template
+
workflow rules
+
step template
+
context
```

Ollama local → nhẹ, free.

Option:

```
model_provider = "ollama" | "openai"
```

# 5️⃣ VECTOR DB LAYER

Bạn cần 3 loại memory:

### A. Project Memory

* Code embeddings
* Docs
* ADR
* Past workflow results

### B. Skill Memory

* Previous reasoning patterns

### C. User Memory

* Preferences

Vector DB use-case:

```
query: "where email validation logic exists?"
→ return relevant file embeddings
→ inject vào skill context
```

Không inject full history.

# 6️⃣ MULTI-THREAD CHAT → BRANCHING MODEL
