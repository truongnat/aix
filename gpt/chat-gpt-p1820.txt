# 🔥 STEP 19 — Redesign Execution Loop

Hiện tại:

```
for step_id in topo_order:
    execute step
```

Giờ ta chuyển sang:

### Level-based execution

1. Compute in_degree map
2. Find nodes with in_degree == 0
3. Execute all in that batch concurrently
4. After batch complete → reduce in_degree
5. Repeat

---

# 🔥 STEP 20 — Implement Async Batch Execution

Trong executor:

Pseudo:

```rust
while ready_nodes not empty:
    sort ready_nodes lexicographically

    let futures = ready_nodes.iter().map(|step_id| async {
        execute_single_step(step_id)
    });

    let results = join_all(futures).await;

    for each completed step:
        update step_results
        update in_degree
```

Use:

```rust
futures::future::join_all
```

---

# 🔒 Determinism Rule

* Always sort ready_nodes before spawn
* Never use unordered HashSet iteration
* Insert results in deterministic order

# 🔥 STEP 22 — ExecutionContext Per Step

⚠️ Critical design point:

Context cannot be shared mutable across parallel tasks.

Solution:

* Clone base context per step
* Do not share &mut ExecutionContext
