# 🚀 PHASE 23 — Opinionated Development Workflows

Mục tiêu:

> Clone repo → có sẵn bộ workflow chuẩn cho solo developer → dùng được ngay.

Không generic. Không enterprise.
Opinionated. Thực dụng. Tối ưu cho 1 người.

---

# 🎯 Triết lý Phase 23

Workflow không phải chỉ là:

LLM → generate code

Mà là:

> Intent → Branch → Plan → Implement → Validate → Review → Merge → Cleanup

Và tất cả phải deterministic + skill-based.

# 🧱 1️⃣ FEATURE WORKFLOW (xương sống hệ thống)

### Mục tiêu

Tạo feature mới từ thread.

### Flow

1. Ensure branch (thread → branch)
2. Load project memory
3. Generate implementation plan (LLM subagent role: Architect)
4. Generate code (LLM subagent role: Implementer)
5. Write files
6. Run validation script
7. Commit
8. Summarize change

### Roles

* Architect
* Implementer
* Validator

### Skills sử dụng

* llm_subagent
* semantic_search
* embed_document
* git_commit
* run_script (test/build)
* echo/report

---

# 🐞 2️⃣ BUGFIX WORKFLOW

Khác feature ở chỗ:

1. Detect failing test / error log
2. Search related code via vector memory
3. Generate fix
4. Run tests
5. Retry if fail (max N times)
6. Commit fix
7. Link to issue

Thêm rule:

* Không refactor quá scope
* Không thêm feature

Bugfix workflow phải bảo thủ hơn feature workflow.

---

# 🔄 3️⃣ REFACTOR WORKFLOW

Flow:

1. Analyze current code
2. Identify smell
3. Propose refactor plan
4. Ensure no behavior change
5. Run full test suite
6. Commit

Thêm constraint:

* No API change unless explicitly allowed
* Must pass all tests

Planner có thể penalize risky operations.

---

# 🔍 4️⃣ REVIEW WORKFLOW

Không modify code.

1. Analyze diff
2. Detect smells / violations
3. Check coding rules
4. Generate review report
5. Optional: auto-fix minor issues

Role:

* Reviewer

Output:

Structured JSON review.

---

# 🔀 5️⃣ MERGE WORKFLOW

1. git_merge_branch
2. analyze_conflicts
3. If conflict:

   * Run conflict resolution subagent
   * Validate build
4. Commit merge
5. Delete feature branch

Đây là nơi bạn tỏa sáng vì đã có isolation + git skills.

---

# 🚀 6️⃣ RELEASE WORKFLOW

1. Ensure clean main
2. Run full validation
3. Generate changelog
4. Tag version
5. Commit release notes

---

# 🧠 Workflow Definition Pattern

Ví dụ feature.yaml nên có dạng:

```yaml
id: feature
description: Implement a new feature from thread intent
roles:
  - architect
  - implementer
  - validator

steps:
  - ensure_branch
  - semantic_search
  - llm_plan
  - llm_implement
  - write_files
  - run_tests
  - git_commit
  - summarize
```

Không hardcode logic trong engine.

Workflow engine chỉ execute steps.

# 🔥 Opinionated Rules

Bạn cần coding rules mặc định như:

* No unused imports
* Must include tests for new feature
* Must not modify unrelated files
* Must update memory index after change
* Must commit with structured message

Ví dụ commit format:

```
type(scope): short summary

- change 1
- change 2
```
