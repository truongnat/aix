# ✅ STEP 2.1 — Define `Skill` Trait (skill/trait.rs)

Yêu cầu:

Trait phải:

* Object-safe
* Thread-safe
* Không phụ thuộc implementation cụ thể
* Không phụ thuộc CLI
* Không phụ thuộc workflow loader

### Interface cần có:

```text
pub trait Skill: Send + Sync {
    fn name(&self) -> &str;
    fn execute(&self, input: &str, ctx: &mut ExecutionContext) -> Result<String>;
}
```

🎯 Quan trọng:

* Không truyền Workflow vào skill
* Không truyền registry vào skill
* Skill chỉ làm 1 việc

# ✅ STEP 2.2 — Define `ExecutionContext` (engine/context.rs)

Context này là runtime state, không phải memory AI.

```text
pub struct ExecutionContext {
    pub workflow_name: String,
    pub step_id: String,
    pub memory: String,
}
```

Hiện tại chỉ cần:

* memory (cumulative)
* workflow_name
* step_id

Không thêm vector DB hay branch gì ở bước này.

# ✅ STEP 2.3 — Define `Workflow` Model (workflow/model.rs)

Workflow là declarative structure.

```text
pub struct Workflow {
    pub meta: WorkflowMeta,
    pub steps: Vec<WorkflowStep>,
}

pub struct WorkflowMeta {
    pub name: String,
    pub domain: Option<String>,
    pub skills: Vec<String>,
}

pub struct WorkflowStep {
    pub id: String,
    pub skill: String,
    pub input: String,
}
```

🎯 Bắt buộc:

* Step phải specify skill
* Không implicit skill list
* Không nested steps

# ✅ STEP 2.4 — Define `SkillRegistry` (engine/registry.rs)

Registry phải:

```text
HashMap<String, Arc<dyn Skill>>
```

Interface:

```text
pub fn register(&mut self, skill: Arc<dyn Skill>);
pub fn get(&self, name: &str) -> Result<Arc<dyn Skill>>;
```

Trả `Arc<dyn Skill>` (clone), không trả reference.

Lý do: tránh lifetime headache sau này.
