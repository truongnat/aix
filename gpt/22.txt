# 🚀 PHASE 22 — Agent Session & Project Runtime Layer

---

# 🎯 Tại sao Phase 22 là bước đúng nhất

Hiện tại bạn có:

* Deterministic executor
* Capability sandbox
* Resource governance
* Isolation backend
* Planner cost-aware

Nhưng bạn chưa có:

* Project-aware execution
* Thread-aware execution
* Git branch orchestration
* LLM sub-agent integration
* `.agent/` layout loader

Nếu không có Phase 22, bạn chỉ có kernel.

Phase 22 = user-facing runtime layer.

---

# 🧱 PHASE 22 — Cấu trúc cụ thể

## 1️⃣ Project Agent Layout System

Chuẩn hóa cấu trúc:

```
.agent/
  workflows/
  skills/
  rules/
  templates/
  memory/
```

Engine phải:

* Auto-load workflows từ project
* Auto-load skill definitions
* Merge project rules với runtime policy
* Validate on startup

Đây là bước để giống Antigravity nhưng deterministic hơn.

---

## 2️⃣ Agent Session Manager

Thêm layer:

```
AgentSession
├── project_root
├── active_thread_id
├── active_branch
├── loaded_workflows
├── vector_index_handle
└── llm_adapter
```

Session sẽ:

* Map chat thread → git branch
* Spawn workflow execution
* Manage context memory
* Track thread isolation

---

## 3️⃣ Thread → Git Branch Orchestrator

Đây là phần cực kỳ quan trọng với solo dev.

Cơ chế:

```
Thread A → feature/login-refactor
Thread B → feature/new-api
Thread C → fix/bug-101
```

Workflow phải:

* Auto create branch
* Execute skills inside branch
* Commit changes
* Merge via merge workflow
* Conflict resolution skill

Bạn đã có isolation và skill execution.
Giờ chỉ cần git skill orchestration.

---

## 4️⃣ LLM Sub-Agent Adapter (Ollama Layer)

Bạn cần skill loại:

```
LLMSubAgentSkill
```

Có:

* role
* prompt template
* tool calling bridge
* structured output schema
* temperature
* model selection (ollama | openai | gemini)

Ollama adapter:

* Local by default
* Optional public provider

Đừng để LLM gọi trực tiếp từ workflow.
Phải wrap thành skill → để governance hoạt động.

---

## 5️⃣ Vector + Graph Memory Layer

Thêm 2 skills:

* `embed_document`
* `semantic_search`

Memory nằm trong:

```
.agent/memory/
```

Dùng:

* SQLite + embedding
  hoặc
* Qdrant local

Workflow sẽ:

1. Query vector
2. Inject context vào LLM skill
3. Generate code

Đây là phần làm cho agent “nhớ project”.

---

## 6️⃣ Script Tooling Layer

Bạn nói đúng:

* validate
* generate skills
* generate workflows
* report
* analyze conflicts

Tất cả phải là skills.

Không hardcode logic trong engine.

Engine chỉ orchestrate.
