# 🟢 PHASE 25 — Structured Telemetry & Trace System

## Mục tiêu

Biến engine từ “correct” → “observable”.

## Cần hoàn thành

### 1️⃣ Structured Step Telemetry

* attempt
* duration_ms
* retry_count
* failure_class
* idempotent_short_circuit
* started_at / finished_at

Persist trong workflow state.

---

### 2️⃣ Deterministic trace_id

Hash:

* workflow definition
* sorted inputs
* engine version

Giúp:

* So sánh run
* Regression testing
* Trace replay

---

### 3️⃣ JSON Trace Export CLI

```
agent workflow trace <id> --json
agent workflow trace <id> --timeline
```

---

### 4️⃣ Timeline Renderer

Human readable execution log.

---

### Completion Criteria

* Telemetry persisted
* JSON export working
* Timeline working
* Tests cover retry + idempotency telemetry

---

# 🟡 PHASE 26 — Sandbox & Security Hardening

Hiện tại engine an toàn về logic.
Chưa an toàn về môi trường thực thi.

## Cần thêm:

### 1️⃣ Environment Isolation

* clear env
* allowlist variables
* block secret leak

---

### 2️⃣ Working Directory Isolation

* per workflow working dir
* prevent accidental overwrite outside repo

---

### 3️⃣ Resource Limits

* CPU limit
* memory cap
* timeout enforcement strict

---

### 4️⃣ Command Policy

* optional allowlist
* deny dangerous commands

---

### Completion Criteria

* run_script không inherit full host env
* test verify isolation
* resource exhaustion không crash engine

---
