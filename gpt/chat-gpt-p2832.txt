# 🔥 STEP 28 — Define ExecutionEvent Model

Tạo file:

```
engine/events.rs
```

Define:

```rust
pub enum ExecutionEvent {
    WorkflowStarted { workflow: String },

    StepScheduled { step_id: String },

    StepStarted { step_id: String },

    StepSucceeded { step_id: String },

    StepFailed {
        step_id: String,
        error: String,
    },

    StepSkipped { step_id: String },

    WorkflowCompleted,
}
```

Không thêm timestamp trước.
Chúng ta thêm sau.

# 🔥 STEP 30 — Emit Events in Executor

Emit tại các điểm:

### Before workflow:

```
WorkflowStarted
```

### When batch scheduling:

```
StepScheduled
```

### Before execute:

```
StepStarted
```

### On success:

```
StepSucceeded
```

### On failure:

```
StepFailed
```

### On skip:

```
StepSkipped
```

### End:

```
WorkflowCompleted
```

---

# 🔒 Determinism Rule

Trong batch:

* Sort ready_nodes
* Emit StepScheduled theo sorted order
* Emit StepStarted theo sorted order
* Sau join_all → emit results theo sorted order

Không emit theo thứ tự future complete.
