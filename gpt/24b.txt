# 📦 PHASE 24 — Completion Checklist (Deterministic Engine)

## I. Core Implementation (BẮT BUỘC)

### 1️⃣ ExecutionEngine v2 (song song)

* [ ] Tạo module mới: `engine/v2/`
* [ ] Implement:

  * `ExecutionEngine`
  * `StepExecutor`
  * `WorkflowInstance`
* [ ] Không ảnh hưởng engine cũ
* [ ] Feature flag:

  ```bash
  AGENT_ENGINE=v2
  ```

**Definition of done:**

* Workflow chạy hoàn toàn qua v2
* Không reuse implicit logic từ v1

---

### 2️⃣ Persistent State Layer

* [ ] `.agent/state/`
* [ ] Atomic write (temp + rename)
* [ ] Lock file (ngăn double run)
* [ ] JSON schema versioning

**Must have:**

```json
{
  "schema": 1,
  "workflow_id": "...",
  "status": "Running",
  "current_step": 2
}
```

**Definition of done:**

* Crash giữa step → resume chính xác
* Không double-execute step đã success

---

### 3️⃣ Failure Semantics Engine

Support:

* [ ] retry
* [ ] timeout
* [ ] on_failure: abort | continue
* [ ] exponential backoff (khuyên nên có)

**Definition of done:**

* Unit test: step fail 2 lần → retry → success
* Timeout kill process đúng cách

---

### 4️⃣ Sandbox Runtime

* [ ] Command whitelist
* [ ] Timeout enforcement
* [ ] Clean environment
* [ ] Exit code normalization
* [ ] Output capture

**Hard requirement để gọi là production-ready.**

**Definition of done:**

* `rm -rf /` không thể chạy
* Command treo → bị kill
* Env biến độc hại không leak

---

### 5️⃣ Policy Engine Integration

* [ ] Pre-step validation
* [ ] Post-step validation (optional)
* [ ] Rule loading từ `.agent/rules`

Policies tối thiểu:

* Branch naming
* Merge rule enforcement
* Runtime safety

**Definition of done:**

* Vi phạm rule → abort trước side-effect

---

## II. CLI & Operational Completeness

### 6️⃣ CLI Commands mới

* [ ] `agent workflow status`
* [ ] `agent workflow resume <id>`
* [ ] `agent workflow abort <id>`
* [ ] `agent workflow list`

**Definition of done:**

* Có thể kill process giữa chừng rồi resume bằng CLI

---

### 7️⃣ Logging & Observability

* [ ] Structured logging (JSON mode)
* [ ] Step timing metrics
* [ ] Failure reason classification

Optional nhưng rất nên có:

* `--verbose`
* `--json` output mode

---

### 8️⃣ Determinism Validation Tests

Phải có integration test:

* [ ] Same workflow + same inputs → same output
* [ ] Crash mid-step → resume idempotent
* [ ] Retry logic deterministic

Nếu thiếu test này → chưa gọi là deterministic engine.

---

## III. Migration Strategy (Quan trọng vì bạn chọn C)

### 9️⃣ Dual Engine Support

* [ ] V1 vẫn chạy được
* [ ] V2 behind flag
* [ ] Regression test chạy cho cả hai

---

### 🔟 Default Switch

Chỉ switch khi:

* [ ] 100% workflows chạy được trên v2
* [ ] Không còn v1-only feature
* [ ] Performance tương đương hoặc tốt hơn

Sau đó:

* [ ] Deprecate v1
* [ ] Remove v1 ở Phase 25

---

# 🧪 IV. Hardening Layer (để gọi là “Completed” thực sự)

Nếu bạn muốn Phase 24 ở mức serious:

### 11️⃣ Idempotency Contracts

Mỗi skill phải định nghĩa:

```rust
fn is_idempotent(&self) -> bool;
fn detect_already_applied(&self) -> bool;
```

Nếu không làm phần này,
resume sẽ luôn tiềm ẩn bug.

---

### 12️⃣ Workflow Schema Versioning

WorkflowDefinition phải có:

```yaml
version: 2
```

Engine phải:

* reject version mismatch
* support upgrade path

---

### 13️⃣ Concurrency Guard

* [ ] Không cho 2 workflow cùng repo chạy song song
* [ ] File lock hoặc PID lock

---

### 14️⃣ Engine Crash Recovery Test

Simulate:

* panic giữa step
* SIGTERM giữa run_script
* disk write interrupted

Nếu resume pass hết → bạn thực sự có deterministic engine.
