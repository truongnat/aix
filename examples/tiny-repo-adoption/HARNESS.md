# HARNESS

## Purpose

Define the adopted harness operating model for a tiny repository.

## Current Status

Example profile.

## Scope

Show the smallest useful profile and goal artifact shape for a health-check planning task.

## Operating Model

Markdown-first planning, review, verification, and memory with no runtime automation.

## Assumptions

- the repository is small
- the team wants a lightweight planning surface

## Unknowns

- whether the repo will need a larger workflow or more roles later

## Human Review

Required before using the profile in a real repository.
