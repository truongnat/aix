# Plan — Form-Filling Guide

How to fill `assets/plan-template.md` so it satisfies the `Output Contract` in `SKILL.md`.

## Source brief

Link or path to the approved `brainstorming` brief this plan implements. If optional `discuss` review happened, also note that disposition. A plan with no traceable brief is a guess, not a plan.

## Tasks

One row per task. Each task must be:

- small enough to verify on its own (see `references/task-sizing.md`)
- paired with a concrete verification step — a command, a check, not "looks right"
- left unchecked (`[ ]`) until `execute` actually runs it and the verification passes

Do not pre-check tasks. The plan is written before any task starts.

## Visual Decisions

Only fill this table if `visualize-question` actually ran. If every decision in this plan is non-visual, leave the table empty — do not invent a row to look thorough.

## Confirmation

Both boxes must be checked by `visualize-result`, not by the author of the plan, before handoff to `execute`.

## Validation

A plan is acceptable only if every task has a verification step and the source brief is a real reference, not a placeholder.
