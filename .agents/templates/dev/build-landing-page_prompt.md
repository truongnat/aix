# Dev Build Landing Page Prompt

Use this template when building a landing page feature end-to-end.

Task context should include:
- product/service name
- target audience and primary pain
- unique differentiator versus alternatives
- core conversion goal (signup/demo/contact/purchase)
- supporting proof (metrics, testimonials, logos, benchmark claims)
- existing stack or file constraints
- visual direction (if any)
- non-functional constraints (SEO/performance/accessibility)

Expected output:
- value narrative map (`problem -> differentiator -> proof -> outcome -> CTA`)
- architecture/spec for landing page sections with copy intent
- implementation-ready file plan (exact paths + create/update intent)
- validation checklist (responsive, accessibility, performance, CTA flow)
- workflow assessment: what is weak in the current process and concrete improvements
